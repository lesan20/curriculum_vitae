### Проект: Мониторинг технического состояния оборудования
#### Руководитель направления 
Инициатор основания направления "Мониторинг и предсказательное обслуживание технического оборудования" в компании. 
Осуществляла развитие данного направления с момента его создания и до релиза продукта -  [модуль для разработки систем мониторинга технического состояния оборудования](https://start.engee.com/monitoring).

  - Руководила рекламной кампанией направления (и впоследствии продукта), включающей создание постов, проведение вебинаров, разработку сайта и другие инициативы (например, [сайт](https://start.engee.com/monitoring), посты в телеграм-каналах [1](https://t.me/exponenta_ru/356), [2](https://t.me/engee_com/244))
  - Разработала концепции продукта, спроектировала пользовательские сценарии, создала макета интерфейса
  - Разработала [установку](https://exponenta.ru/events/predskazanie-otkazov-v-promishlenosi) для демонстрации технологии мониторинга и предсказательного обслуживания, включая проектирование установки и выбор компонентов. Установка включает асинхронные двигатели и набор датчиков. Выполнена сборка установки и организована система сбора данных. Созданы физические неисправности для формирования выборки. Обучены различные модели для демонстрации возможностей предсказательного обслуживания, включая:
      - Обнаружение аномалий с использованием классических методов машинного обучения и Autoencoder.
      - Классификацию неисправностей с применением bi-LSTM и CNN на спектрограммах.
      - Классификацию аномалий на ранней стадии с использованием вариационного автоэнкодера (VAE).
  - Разработка архитектуры пилотного варианта продукта (метод отображения С4) и реализация, использованный стек: Python, PostgreSQL, DataLens, Docker, sklearn, Keras, git CI/CD.
  - Презентация продукта клиентам ([пример](https://exponenta.ru/events/engee-monitoring?utm_source=website&utm_medium=engee_event&utm_campaign=25.04_veb&clckid=3d6bf8af))

#### TeamLead команды разработки
Руководство процессом разработки модуля Engee.Мониторинг (стационарная команда 2 чел. + частичная занятость 2 чел.).  
Engee.Мониторинг - Модуль для разработки систем мониторинга технического состояния оборудования в составе среды разработки Engee (совместим с MATLAB, Python).  
Стек: Python, Elasticsearch, Grafana, Docker, sklearn, Keras, Pytorch.  
AI: обработка Time Series, SVM, XGBoost trees, LSTM, Autoencoder, VAE.

  - Осуществляла планирование фичей продукта, управление командой, вела мониторинг выполнения задач и отчетность
  - Реализовала примеры пайплайнов разработки мониторинга на примере данных двигателей и трансформаторов
  - Разработала алгоритм непрерывной генерации данных реалистичного поведения
  - Разработала шаблоны панелей мониторинга

### Проект "Алгоритм управления балансом газа в газотранспортной сети"
Фриланс-проект. Реализация теоретических идей научной группы в рамках сотрудничества с компанией ПАО "Газпром". 
  - Обучила культуре ведения разработки (git, тестирование, автоматизация)
  - Автоматизировала пайплайн работы с данными от момента выгрузки от заказчика до получения результата по объекту (очистка выбросов и некорректных данных, обнаружение сбоев при сборе данных, агрегация данных и создание датасетов для обучения моделей)
  - Разработала алгоритм обнаружения и классификации аномалий в работе объектов (газораспределительные и газоизмерительные станции) на основе LSTM, сформировала схемы обучения, дообучения (предотвращение дрифта моделей) и использования алгоритма на новых объектах
  - Реализовала и оптимизировала алгоритм управления балансом газа в газотранспортной станции на основе предоставленной блок-схемы
  - Алгоритм повысил эффективность управления баланса газа на 30 % согласно тестам
